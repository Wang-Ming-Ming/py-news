# Serenity 卡点分析

把 **Serenity (@aleabitoreddit)** 的公开方法论落地为可执行的 AI 工作流：**单事件/趋势 → 画链 → 找最稀缺一环 → 反推受益标的 → 七步评估 → 反证**。

> 本 skill 产出研究框架，**不是**投资建议。所有结论必须标注证据等级与不确定性。

## 何时使用

- 用户给一条新闻、财报片段、政策/订单信号，问「卡点在哪、谁受益」
- 用户想复刻 Serenity 式「未知瓶颈」研究，而非追共识龙头
- 用户已有候选股，想用七步法做 chokepoint 验证
- 产业链瓶颈分析、卡点实验室式深挖任务

## 何时不用

- 用户只要技术面/动量/短线买卖点（本 skill 是基本面+产业链）
- 用户已有完整交易计划，只需格式化输出
- 输入是纯叙事、无物理/产能/供应链约束可验证

## 核心原则（Doctrine）

1. **做「未知的瓶颈」，不做「共识赢家」** — 大票通常没有 re-rating 空间
2. **读财报看「谁卡谁」**，不要只看「需求在涨」
3. **永远准备物理替代路径** — 每个卡点都要问「什么技术/材料能绕开它」
4. **用 % 验证里程碑**，不用「翻倍」这类美元目标
5. **一手信源优先** — 10-K/电话会/专利/产能指引 > 二手研报叙事
6. **先锁卡点，再出票** — 禁止从新闻直接跳到「看起来沾边」的票

## 工作流

按顺序执行；任一步结论不足时，**联网搜索补证**后再继续，并在输出中记录「搜了什么、否定过什么」。

```
进度:
- [ ] 0. 锚定事件与 megatrend
- [ ] 1. 物理约束
- [ ] 2. 画链（自下而上）
- [ ] 3. 一手信号
- [ ] 4. 锁定 1–3 个卡点
- [ ] 5. 反推受益标的
- [ ] 6. 估值不对称 / 持股 / 催化 / 风险（Step 4–7）
- [ ] 7. 反证与置信度
```

### Step 0 · 锚定

明确：

- **触发事件**（原文摘要，≤3 句）
- **Megatrend**（已确认的大趋势，非猜测）
- **市场范围**（美股 / A 股 / 其他；默认跟随用户语境）
- **时间窗**（短期催化 / 中期瓶颈 / 长期结构）

若 megatrend 本身未经验证，先搜索再进入 Step 1。

### Step 1 · 物理约束

回答：**需求是被物理规律/产能/工艺/认证倒逼，还是纯叙事？**

| 信号 | 倾向真卡点 |
|------|-----------|
| 扩产周期以年计、良率/认证难复制 | 是 |
| 设备/材料/衬底/封装有明确 duopoly/ oligopoly | 是 |
| 只有「AI 很火」「政策利好」、无供给约束 | 否（警告用户） |

输出：`物理约束：强/中/弱` + 一句依据。

### Step 2 · 画链（自下而上）

从 **原料 → 衬底/材料 → 设备 → 外延/制造 → 芯片 → 模块 → 系统 → 终端** 画产业链树。

要求：

- 每层 1 句：**这一层产出什么、谁需要它**
- 标出 **疑似稀缺节点**（先 3–5 个，不锁定）
- 用户若限定区间（如「只看光模块到交换机」），收窄链条

输出：Markdown 树或表格；见 [reference.md](reference.md) 模板。

### Step 3 · 一手信号

对每个疑似节点，找 **至少一类** 一手证据：

- 财报/电话会：lead time、backlog、capacity、capex、客户集中度
- 专利/论文/标准：技术路线是否被锁定
- 价格/交期：现货、合约、行业报价
- 监管/出口/认证：准入壁垒

**信号反指**：CEO 说「我们被 xx 卡住了」→ 往往指向 chokepoint，而非自家叙事。

证据不足时写 `-`，**不要编造**。

### Step 4 · 锁定卡点

用 **五因子模型** 对疑似节点打分（见 [reference.md](reference.md)）：

1. 确定需求（megatrend 已验证）
2. 供给约束（复制慢、壁垒高）
3. 市场低关注（非共识龙头）
4. 价值捕获（定价权/份额）
5. 近端催化（财报、量产、认证、订单）

**锁定 1–3 个卡点**，每个写：

- 环节名称
- 为何最稀缺（与上下游对比）
- 物理替代路径（必须写）
- 置信度：高/中/低

### Step 5 · 反推受益标的

规则：

- 标的必须 **踩在已锁定卡点环节**，不能「赛道相关」
- 每只票写 **踩点理由**（1–2 句）：公司在该环节的份额/角色/唯一性
- 区分 **直接受益** vs **间接受益**；默认只推荐直接受益
- A 股：优先 6 位代码 + 简称；美股：ticker + 公司名
- 无合适标的时明确说「卡点成立但缺少可交易映射」，不要硬凑
- **【硬性】踩点核实（六件套）**：写完每只票的踩点理由后，**必须**用以下六类一手验证项核完，**任何一项不达标就降级或剔除**。注意：前 4 件查"公司是否在该卡点有业务"，第 5-6 件查"业绩是否真兑现"——后者是更常见的陷阱来源。
  
  **【卡点归属层】信源核（前 4 件）**
  1. **公司投资者互动平台**（深交所互动易/上证 e 互动/公司 IR 邮箱）—— 最常出现"我司尚未开展 XX 业务"的官方表态
  2. **招股书 / 募集说明书 / 重大资产重组报告书** —— 业务描述章节
  3. **季报 / 年报"经营情况讨论与分析"** —— 主营业务分行业/分产品收入构成
  4. **官方公告**（巨潮资讯 / 公司官网新闻）—— 重大项目、扩产、战略合作
  
  **【业绩兑现层】业绩+估值核（后 2 件）**
  5. **业绩兑现验证**（最关键反陷阱）—— 查过去 3-4 个季度季报/年报，**该卡点业务的实际收入增速**：
     - 必须能在「主营业务分行业/分产品」表里找到该业务，**且增速 ≥ 公司整体平均增速**
     - 占比 < 5% 的业务 → 降级（盘子太小，业绩弹性不显著）
     - 增速 < 公司整体平均 → 降级（实际没吃到卡点红利）
     - 公司整体业绩负增长时，该业务**仍正增长且显著高于行业** → 才是真受益
     - 反例：见 [reference.md](reference.md) "业绩兑现陷阱"章节（石英股份案例）
  6. **估值合理性检查** —— 业绩增速 vs PE 是否匹配：
     - 业绩同比 -10% 以上 + PE(TTM) > 50x → **警示**，可能已严重透支
     - 业绩增速 < PE 的 1/2 → **警示**，赔率不足
     - 业绩正增长但 PE > 100x（无强催化）→ 警示容错率极低
- **市场标签 ≠ 实际踩点**。常见陷阱：
  - "HBM 封测三强" / "存储封测龙头" / "核心供应商" / "国家队" 等市场笼统说法，只做参考、不做依据
  - 必须看到公司自己说"我司是 XX 环节的 XX 角色"，才算踩点
  - 反例：见 [reference.md](reference.md) "市场标签陷阱"章节
- 区分「现在卡点」（公司当前已开展并产生收入）和「未来卡点」（研发布局、产能建设中、储备阶段）—— 只有现在卡点可以放进 Step 5 受益标的；未来卡点放 Step 7 反证或观察信号

### Step 6 · 七步评估（对每只候选票）

对 Step 5 的每只票，快速过 Serenity 4–7 步：

| 步骤 | 问什么 |
|------|--------|
| 4 估值不对称 | 市值 vs 卡点捕获空间；是否已被定价 |
| 5 持股结构 | 机构是否已重仓；是否还有 retail→institution 空间 |
| 6 催化时间表 | 未来 3–12 个月可验证的 **% 里程碑**（产能利用率、毛利率、订单同比等） |
| 7 风险分层 | 什么能 **杀死** thesis（替代技术、稀释、客户自研、周期下行） |

### Step 7 · 反证与交付

必须包含：

- **最强反证**（至少 2 条）
- **什么新信息会推翻结论**
- **推理回放摘要**（搜过啥、否定过啥假设）

## 输出格式

默认使用 [reference.md](reference.md) 中的 **实验室报告模板**。

简短模式（用户只要结论）：Executive Summary ≤200 字 + 卡点表 + 候选票表 + 反证。

## 交互模式

**完整实验室**（默认）：逐步推进，每步可暂停等用户「追问 / 换条链 / 深挖某环节 / 锁定或否定」。

**快速扫描**：用户明确说「快速」时，可合并 Step 1–4，但仍需画链 + 反证，不可跳过物理约束。

## 质量门禁

交付前自检：

- [ ] 卡点是否具体到产业链 **某一环节**，而非「半导体」「AI」
- [ ] 每只票的受益逻辑是否 **经过该卡点**，而非赛道标签
- [ ] 是否写了 **物理替代路径**
- [ ] 是否区分 **事实 / 推断 / 未知**
- [ ] 是否包含 **非投资建议** 声明
