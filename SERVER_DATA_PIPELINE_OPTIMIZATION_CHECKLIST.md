# 服务器数据管线优化与部署清单

记录日期：2026-06-20

目标：今天完成“服务器自动采集并通过接口提供完整数据，本地三个推荐 skill 取数，本地 Codex 在聊天窗口完成全部分析”的第一版，并部署到服务器。

> 本文件是本次服务器优化与部署的最终执行依据。若它与此前文档中“服务器判断主线、公告风险、硬逻辑、候选池或推荐分数”等内容冲突，以本文件为准。

## 1. 最终架构

```text
上游新闻 / 公告 / 市场数据
            ↓
服务器：采集、清洗、去重、保存、客观计算、数据健康、只读 API
            ↓
本地客户端：自动分页、下载、续传、校验、缓存
            ↓
本地 skill 脚本：按各自策略需要整理客观数据
            ↓
本地 Codex 聊天：分析、判断、选股、排名、仓位、买卖计划
```

最终使用方式不变：用户继续在本地 Codex 聊天窗口里运行：

- `morning_stock_picker`
- `overnight_stock_picker`
- `short_term_trend_trader`

服务器现在和以后都不运行 AI，不需要任何 AI 密钥、模型、推理服务或自动推荐任务。未来即使扩建数据源，服务器职责也始终只限于数据自动采集、客观整理、保存和接口提供。

## 2. 服务器职责硬边界

### 2.1 服务器允许做

- [ ] 按真实 A 股交易日历调度任务。
- [ ] 按来源限制安全采集新闻、公告和市场数据。
- [ ] 保存原始字段、原文链接、来源时间和采集时间。
- [ ] 数据去重、格式统一、压缩、索引和归档。
- [ ] 保存竞价、早盘和尾盘关键时间点行情。
- [ ] 计算纯数学指标：均线、涨跌幅、量比、换手、振幅、位置等。
- [ ] 统计客观数量：涨停数、炸板数、上涨/下跌家数、来源直接提供的行业/概念数据。
- [ ] 检查数据是否缺失、过期、重复或采集失败。
- [ ] 通过只读接口提供完整数据和按条件查询能力。

### 2.2 服务器禁止做

- [ ] 不判断市场主线。
- [ ] 不判断初动、确认、高潮、轮动或退潮。
- [ ] 不判断公告利好、利空或风险等级。
- [ ] 不判断新闻强弱、硬逻辑或产业卡点。
- [ ] 不推断没有官方依据的产业链关系。
- [ ] 不判断企稳、吸筹、承接、出货或抛压。
- [ ] 不生成候选池、推荐分数或股票排名。
- [ ] 不预测下一交易日资金路径。
- [ ] 不给出买入、卖出、止损或仓位建议。

### 2.3 客观字段命名

服务器可以输出：

- `return_5d`
- `return_10d`
- `return_15d`
- `ma5`
- `ma10`
- `ma20`
- `volume_ratio_5d`
- `upper_shadow_ratio`
- `range_position_15d`
- `price_change_1430_1450`
- `amount_change_1430_1450`

服务器禁止输出：

- `is_stabilized`
- `distribution_risk`
- `main_theme_stage`
- `hard_logic_score`
- `next_day_buy_score`
- `recommended_candidate`

## 4. 数据协议与版本

- [ ] 所有时间统一为 `Asia/Shanghai`。
- [ ] 同时保存上游数据时间 `source_time` 和本地采集时间 `collected_at`。
- [ ] 统一证券代码为六位字符串。
- [ ] 明确价格、成交量、成交额和市值单位。
- [ ] 保留上游原始字段，标准化字段不能覆盖原始值。
- [ ] 每份不可变市场快照生成唯一 `snapshot_id`。
- [ ] 每次数据发布生成唯一 `dataset_version`。
- [ ] 每个数据集记录 `expected_count`、`actual_count`、`checksum` 和 `is_complete`。
- [ ] 最新指针只指向完整有效数据，失败数据不能覆盖有效数据。

## 6. 安全采集与防限制

### 6.1 每个来源独立配置

- [ ] 最小请求间隔。
- [ ] 最大并发数，默认单域名 `1-2`。
- [ ] 连接和读取超时。
- [ ] 最大重试次数。
- [ ] 指数退避时间。
- [ ] 连续失败熔断时间。
- [ ] 最近成功时间和最近错误。

### 6.2 采集纪律

- [ ] 只做增量采集，不重复请求完整历史数据。
- [ ] 同一域名的多个接口错开 `20-60` 秒。
- [ ] 复用连接，避免频繁建立新连接。
- [ ] 收到 `429` 或 `403` 后立即暂停并延长重试间隔。
- [ ] 不使用代理轮换绕过来源限制。
- [ ] 单一来源失败不能中止其他来源。
- [ ] 健康检查只读取本地状态，不请求上游。
- [ ] 高频行情优先使用稳定、合规的数据接口。

### 6.3 第一版建议频率

- [ ] 财联社：交易时段每 `3-5` 分钟。
- [ ] 东方财富全球快讯：每 `5-10` 分钟。
- [ ] 巨潮公告：交易时段每 `15-30` 分钟；`15:00-22:00` 每 `10-15` 分钟。
- [ ] 发改委：每 `30-60` 分钟；周末降低频率。
- [ ] 全市场普通行情：仅在规定关键时间生成快照。
- [ ] 全市场日线：每个交易日收盘后增量更新一次。

- [ ] 新闻索引使用游标分页，每页 `200` 条。

- [ ] 新闻和公告正文通过 ID 按需获取。

### 12.2 一致性要求

- [ ] 禁止使用会在翻页过程中漂移的实时 offset 分页。
- [ ] 最后一页必须返回 `next_cursor = null`。
- [ ] 下载完成后验证条数和 checksum。
- [ ] 未完成的数据不能替换本地有效缓存。
