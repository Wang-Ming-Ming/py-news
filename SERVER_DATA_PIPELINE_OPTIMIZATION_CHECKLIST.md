## 1. 最终架构

```text
上游新闻 / 公告 / 市场数据
            ↓
服务器：采集、清洗、去重、保存、客观计算、数据健康、只读 API
            ↓
本地客户端：自动分页、下载、续传、校验、缓存
            ↓
本地 skill 脚本：按各自策略需要整理客观数据
            ↓
本地 Codex 聊天：分析、判断、选股、排名、仓位、买卖计划
```

最终使用方式不变：用户继续在本地 Codex 聊天窗口里运行：

- `morning_stock_picker`
- `overnight_stock_picker`
- `short_term_trend_trader`

服务器现在和以后都不运行 AI，不需要任何 AI 密钥、模型、推理服务或自动推荐任务。未来即使扩建数据源，服务器职责也始终只限于数据自动采集、客观整理、保存和接口提供。

## 2. 服务器职责硬边界

### 2.1 服务器允许做

- [x] 按真实 A 股交易日历调度任务。
- [x] 按来源限制安全采集新闻、公告和市场数据。
- [x] 保存原始字段、原文链接、来源时间和采集时间。
- [x] 数据去重、格式统一、压缩、索引和归档。
- [x] 原始新闻、公告和市场快照永久按交易日期保留，不因 API 查询窗口而删除。
- [x] 保存竞价、早盘和尾盘关键时间点行情。
- [x] 计算纯数学指标：均线、涨跌幅、量比、换手、振幅、位置等。
- [x] 统计客观数量：涨停数、炸板数、上涨/下跌家数、来源直接提供的行业/概念数据。
- [x] 检查数据是否缺失、过期、重复或采集失败。
- [x] 通过只读接口提供完整数据和按条件查询能力。

### 2.2 服务器禁止做

- [x] 不判断市场主线。
- [x] 不判断初动、确认、高潮、轮动或退潮。
- [x] 不判断公告利好、利空或风险等级。
- [x] 不判断新闻强弱、硬逻辑或产业卡点。
- [x] 不推断没有官方依据的产业链关系。
- [x] 不判断企稳、吸筹、承接、出货或抛压。
- [x] 不生成候选池、推荐分数或股票排名。
- [x] 不预测下一交易日资金路径。
- [x] 不给出买入、卖出、止损或仓位建议。

## 4. 数据协议与版本

## 6. 安全采集与防限制

### 6.1 每个来源独立配置

- [x] 最小请求间隔。
- [x] 最大并发数，默认单域名 `1-2`。
- [x] 连接和读取超时。
- [x] 最大重试次数。
- [x] 指数退避时间。
- [x] 连续失败熔断时间。
- [x] 最近成功时间和最近错误。

### 6.2 采集纪律

- [x] 只做增量采集，不重复请求完整历史数据。
- [x] 不同来源首次启动错开 `20` 秒；同域请求串行且至少间隔 `1` 秒。
- [x] 复用连接，避免频繁建立新连接。
- [x] 收到 `429` 或 `403` 后立即暂停并延长重试间隔。
- [x] 不使用代理轮换绕过来源限制。
- [x] 单一来源失败不能中止其他来源。
- [x] 健康检查只读取本地状态，不请求上游。
- [x] 高频行情优先使用稳定、合规的数据接口。

### 6.3 第一版建议频率

- [x] 财联社：交易时段每 `3-5` 分钟。
- [x] 东方财富全球快讯：每 `5-10` 分钟。
- [x] 巨潮公告：交易时段每 `15-30` 分钟；`15:00-22:00` 每 `10-15` 分钟。
- [x] 发改委：每 `30-60` 分钟；周末降低频率。
- [x] 全市场普通行情：仅在规定关键时间生成快照。
- [x] 全市场日线：每个交易日收盘后增量更新一次。
- [x] 首次部署先回填最近 20 个已完成交易日的真实全市场日线，至少 15 日完整才对本地标记为可用。
- [x] 历史回填按股票限速、保存检查点并支持中断续跑，不用当前快照冒充历史快照。
- [x] 服务器原始数据保留期设为 `0`（永久）；API 和本地 skill 默认查询窗口单独设为最近 `15` 天。
